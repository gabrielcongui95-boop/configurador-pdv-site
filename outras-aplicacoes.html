Monitor do Pedidos WEB

Solução composta pelo agente local de segundo plano e um Dashboard unificado, projetada para gerenciar, analisar e monitorar a estabilidade da aplicação Pedidos Web, fornecendo controle para o time de suporte ter o controle de todas as lojas que estão utilizando a aplicação e acompanhar a estabilidade, para que a loja não tenha falha na recepção de pedidos.

Monitoramento Varredura contínua em segundo plano, assegurando uma interface responsiva e livre de travamentos
Aiven Nuvem Persistência centralizada em infraestrutura de banco de dados protegida por criptografia SSL ativa
Auto Restart mata a tarefa travada, limpa cache técnico e reinicia o Pedidos WEB

Fluxo de Funcionamento

1. Inicialização

O Monitor do Pedidos Web.exe inicia junto com o sistema operacional de forma nativa e silenciosa.

2. Coleta de Logs

Varre localmente a atividade e integridade do Pedidos WEB a cada 15 segundos sem gerar gargalos de hardware.

3. Sincronização Nuvem

Despacha os batimentos operacionais diretamente para o cluster para consolidação imediata dos dados de rede.

4. Consolidação

O Dashboard renderiza em tempo real as informações recebidas para triagem e filtragem de logs na tela do suporte.

5. Análise de Histórico

Permite abrir relatórios individuais por loja diretamente no Dashboard, puxando filtros temporais em memória.

Prévia de Interface do analista: Visualizando os detalhes da loja

Status de Monitoramento no Dashboard

🟢 ONLINE

O computador do cliente está ligado, o agente local em execução e o serviço do Pedidos WEB operando perfeitamente e comunicando dados sem latência.

🔴 OFFLINE

O computador do cliente está ligado e ativo, porém o serviço do Pedidos WEB travou, parou de responder ou não está gerando atualizações de logs.

⚫ DESLIGADO

Identifica de forma inteligente quando o computador do cliente está totalmente desligado, diferenciando quedas de aplicação de encerramentos locais de expediente.

Prévia de Interface do analista: Dashboard Centralizado de Lojas sendo monitoradas

Gerenciamento Avançado

Filtro Temporal Dinâmico

O drop-down do painel mapeia os logs consolidados e exibe exclusivamente os dias com intercorrências reais. A análise é executada em memória, reduzindo o tráfego e requisições ao banco de produção.

Limpa histórico

Interface administrativa equipada com caixas de seleção que permitem limpar históricos antigos, facilitando uma análise com os dados mais recentes enviados.

Rastreio de AppDomain

O monitor local realiza uma inspeção de baixo nível para rastrear e identificar dinamicamente o ID e o escopo do AppDomain correto da aplicação. Isso evita leituras cruzadas e garante o monitoramento exclusivo da instância operacional activa do cliente.

Limpeza na Assembly Cache

Mecanismo interno automatizado que localiza e apaga os dados temporários e obsoletos retidos na pasta assemblycache. Essa rotina previne o travamento por corrupção de assemblies carregados e otimiza a estabilidade na inicialização do PDV.
